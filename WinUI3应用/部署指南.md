## 开发环境准备

### 1. 安装Visual Studio 2022

#### 下载安装

1. 访问 [Visual Studio 下载页面](https://visualstudio.microsoft.com/downloads/)
2. 下载 Visual Studio 2022 Community（免费）或 Professional/Enterprise
3. 运行安装程序

#### 选择工作负载

在安装程序中选择以下工作负载：

- **.NET 桌面开发**
  - .NET 8 SDK
  - .NET 桌面开发工具
  
- **通用 Windows 平台开发**
  - Windows 10/11 SDK
  - Windows App SDK C# 模板

#### 安装单个组件

在"单个组件"选项卡中搜索并安装：

- Windows App SDK (最新版本)
- Windows 10 SDK (10.0.19041.0 或更高)
- .NET 8.0 SDK

### 2. 验证安装

打开 Visual Studio 2022，创建新项目，搜索"WinUI"，应该能看到"空白应用，打包的WinUI 3"模板。

## 项目配置

### 1. 克隆项目

```bash
git clone https://github.com/your-repo/hmnl-desktop.git
cd hmnl-desktop/WinUI3应用
```

### 2. 配置API地址

编辑 `HmnlDesktop/appsettings.json`：

```json
{
  "ApiBaseUrl": "https://your-api-url.com",
  "SupabaseUrl": "https://your-project.supabase.co",
  "SupabaseKey": "your-supabase-key",
  "SignalRHubUrl": "https://your-api-url.com/hubs/chat"
}
```

#### 获取Supabase配置

1. 访问 [Supabase](https://supabase.com/)
2. 创建新项目或选择现有项目
3. 进入项目设置 → API
4. 复制 Project URL 和 anon public key

#### 获取API地址

1. 部署后端API服务
2. 获取API的公开访问地址

### 3. 配置应用信息

编辑 `HmnlDesktop/Package.appxmanifest`：

```xml
<Properties>
  <DisplayName>HMNL直播讨论站</DisplayName>
  <PublisherDisplayName>HMNL Team</PublisherDisplayName>
  <Description>HMNL直播讨论站桌面应用</Description>
  <Logo>Assets\StoreLogo.png</Logo>
</Properties>
```

## 本地开发

### 1. 打开项目

1. 启动 Visual Studio 2022
2. 选择"打开项目或解决方案"
3. 导航到项目目录并选择 `HmnlDesktop.sln`

### 2. 选择目标平台

在工具栏中选择目标平台：

- **x64** - 64位Windows系统（推荐）
- **ARM64** - ARM64设备（如Surface Pro X）

### 3. 运行项目

按 **F5** 或点击"开始"按钮运行项目。

### 4. 调试技巧

#### 断点调试

1. 在代码行号左侧点击设置断点
2. 按F5启动调试
3. 程序会在断点处暂停

#### 实时可视化树

1. 调试时选择"调试" > "窗口" > "实时可视化树"
2. 查看和修改运行时的UI元素

#### 输出窗口

1. 选择"调试" > "窗口" > "输出"
2. 查看调试信息和错误

## 构建发布

### 1. 开发构建

```bash
cd HmnlDesktop
dotnet build
```

### 2. 发布构建

#### 发布为独立可执行文件

```bash
dotnet publish -c Release -r win-x64 --self-contained
```

#### 发布为依赖框架的可执行文件

```bash
dotnet publish -c Release -r win-x64 --no-self-contained
```

### 3. 构建配置

| 配置 | 说明 |
|------|------|
| Debug | 调试版本，包含调试信息，体积大 |
| Release | 发布版本，经过优化，体积小 |

### 4. 运时标识符 (RID)

| RID | 说明 |
|-----|------|
| win-x64 | Windows 64位 |
| win-x86 | Windows 32位 |
| win-arm64 | Windows ARM64 |

## MSIX打包

### 1. 使用Visual Studio打包

#### 创建新包

1. 右键项目 > 发布
2. 选择"MSIX打包"
3. 点击"下一步"

#### 配置包信息

- **显示名称**: HMNL直播讨论站
- **发布者**: CN=YourCompany
- **版本**: 1.0.0.0
- **包名称**: HmnlDesktop

#### 选择目标平台

- **x64** - 64位Windows系统
- **ARM64** - ARM64设备

#### 创建包

点击"创建"按钮，生成MSIX包。

### 2. 使用命令行打包

```bash
cd HmnlDesktop
dotnet build -c Release
```

然后使用 `MakeAppx.exe` 工具创建MSIX包。

### 3. 签名

#### 创建测试证书

```powershell
New-SelfSignedCertificate -Type Custom -Subject "CN=HMNL Desktop" -CertStoreLocation "Cert:\LocalMachine\My"
```

#### 签名MSIX包

```powershell
Add-AppxPackage -Path "HmnlDesktop_1.0.0.0_x64.appx" -SignPath "certificate.pfx"
```

## 侧载安装

### 1. 启用侧载

#### Windows 10

1. 打开"设置" > "更新和安全"
2. 选择"开发者选项"
3. 启用"侧载应用"

#### Windows 11

1. 打开"设置" > "隐私和安全"
2. 选择"开发者"
3. 启用"侧载应用"

### 2. 安装依赖

#### 安装Windows App Runtime

下载并安装 [Windows App Runtime](https://aka.ms/windowsappsdk)

### 3. 安装应用

#### 使用PowerShell

```powershell
Add-AppxPackage -Path "HmnlDesktop_1.0.0.0_x64.appx"
```

#### 双击安装

直接双击MSIX包文件，按照提示安装。

### 4. 卸载应用

#### 使用PowerShell

```powershell
Get-AppxPackage *HmnlDesktop* | Remove-AppxPackage
```

#### 使用设置

1. 打开"设置" > "应用"
2. 找到"HMNL直播讨论站"
3. 点击"卸载"

## 常见问题

### Q1: 构建失败怎么办？

**A:** 检查以下几点：
1. 确认已安装.NET 8 SDK
2. 确认已安装Windows App SDK
3. 清理并重新构建项目
4. 查看构建日志中的错误信息

### Q2: 如何处理依赖问题？

**A:** 运行以下命令：
```bash
dotnet restore
```

### Q3: 如何减小应用体积？

**A:** 使用以下方法：
1. 发布为依赖框架的可执行文件（需要用户安装.NET运行时）
2. 启用裁剪发布：`dotnet publish -c Release -r win-x64 --self-contained -p:PublishTrimmed=true`
3. 压缩MSIX包

### Q4: 如何支持多语言？

**A:** 添加资源文件：
1. 在项目中创建 `Strings` 文件夹
2. 为每种语言创建子文件夹（如 `en-US`、`zh-CN`）
3. 在每个文件夹中添加 `resources.resw` 文件
4. 在XAML中使用 `x:Uid` 引用资源

### Q5: 如何实现自动更新？

**A:** 使用以下方法：
1. 集成Windows应用商店更新API
2. 或使用第三方更新框架（如Squirrel.Windows）
3. 定期检查更新并提示用户下载

### Q6: 如何调试已安装的应用？

**A:** 使用以下方法：
1. 附加到进程：调试 > 附加到进程
2. 使用事件跟踪：Windows事件查看器
3. 启用详细日志：在appsettings.json中设置日志级别
