## 安装问题

### 问题1: 无法打开项目

**症状**: 在Visual Studio中打开项目时出现错误

**解决方案**:

1. **检查Visual Studio版本**
   - 确保使用Visual Studio 2022 (17.8或更高版本)
   - 检查"帮助" > "关于Microsoft Visual Studio"

2. **安装必要的工作负载**
   - 打开Visual Studio Installer
   - 修改安装，确保已选择".NET 桌面开发"工作负载
   - 确保已安装Windows App SDK

3. **更新Visual Studio**
   - 打开Visual Studio Installer
   - 点击"更新"
   - 安装最新更新

### 问题2: NuGet包还原失败

**症状**: 还原NuGet包时出现错误

**解决方案**:

1. **清理NuGet缓存**
   ```bash
   dotnet nuget locals all --clear
   ```

2. **手动还原包**
   ```bash
   dotnet restore
   ```

3. **检查NuGet源**
   - 打开Visual Studio
   - 工具 > 选项 > NuGet包管理器 > 包源
   - 确保nuget.org在列表中

### 问题3: Windows App SDK未找到

**症状**: 编译时提示找不到Windows App SDK

**解决方案**:

1. **安装Windows App SDK**
   - 下载 [Windows App SDK](https://aka.ms/windowsappsdk)
   - 运行安装程序

2. **验证安装**
   - 打开Visual Studio
   - 创建新项目，搜索"WinUI 3"
   - 确认能看到WinUI 3模板

## 构建问题

### 问题1: 编译错误

**症状**: 构建时出现编译错误

**解决方案**:

1. **清理并重新构建**
   - 在Visual Studio中选择"生成" > "清理解决方案"
   - 然后选择"生成" > "重新生成解决方案"

2. **检查错误列表**
   - 查看"错误列表"窗口
   - 双击错误跳转到代码位置
   - 根据错误信息修复问题

3. **常见编译错误及解决**

   **错误**: "找不到类型或命名空间"
   - 解决: 检查using语句，确保引用了正确的命名空间

   **错误**: "当前上下文中不存在名称"
   - 解决: 检查变量/方法名拼写，确保在作用域内

   **错误**: "无法将类型X隐式转换为类型Y"
   - 解决: 使用显式转换或检查类型兼容性

### 问题2: XAML编译错误

**症状**: XAML文件编译时出现错误

**解决方案**:

1. **检查XAML语法**
   - 确保所有标签正确闭合
   - 检查属性名称和值
   - 验证数据绑定语法

2. **常见XAML错误及解决**

   **错误**: "无法找到名为XXX的资源"
   - 解决: 确保在App.xaml或页面资源中定义了该资源

   **错误**: "属性XXX无法识别"
   - 解决: 检查属性名称拼写，确认控件支持该属性

   **错误**: "绑定路径无效"
   - 解决: 检查绑定路径，确保属性存在

### 问题3: 资源文件缺失

**症状**: 运行时提示找不到资源文件

**解决方案**:

1. **检查文件属性**
   - 在解决方案资源管理器中右键文件
   - 属性 > 生成操作
   - 设置为"内容"
   - 复制到输出目录: "如果较新则复制"

2. **验证文件路径**
   - 确保路径大小写正确
   - 使用相对路径而非绝对路径

## 运行时问题

### 问题1: 应用启动失败

**症状**: 点击运行后应用立即崩溃

**解决方案**:

1. **检查异常**
   - 在App.xaml.cs中添加全局异常处理
   ```csharp
   public App()
   {
       InitializeComponent();
       UnhandledException += OnUnhandledException;
   }

   private void OnUnhandledException(object sender, Microsoft.UI.Xaml.UnhandledExceptionEventArgs e)
   {
       e.Handled = true;
       // 记录异常信息
   }
   ```

2. **检查事件查看器**
   - 打开"事件查看器"
   - Windows日志 > 应用程序
   - 查找与应用相关的错误事件

### 问题2: API调用失败

**症状**: 应用无法连接到API

**解决方案**:

1. **检查网络连接**
   - 确保设备已连接到互联网
   - 尝试在浏览器中访问API地址

2. **验证API配置**
   - 检查appsettings.json中的ApiBaseUrl
   - 确保URL格式正确（包含http://或https://）

3. **检查防火墙设置**
   - 确保防火墙允许应用访问网络
   - 添加应用到防火墙例外

4. **启用详细日志**
   - 在ApiService中添加日志输出
   - 记录请求和响应详情

### 问题3: 数据绑定不工作

**症状**: UI不更新数据

**解决方案**:

1. **检查ViewModel**
   - 确保属性使用[ObservableProperty]特性
   - 确保命令使用[RelayCommand]特性

2. **检查绑定模式**
   - 确保使用正确的绑定模式（OneWay/TwoWay）
   - 对于需要更新的属性使用Mode=TwoWay

3. **验证DataContext**
   - 确保页面的DataContext设置为ViewModel
   - 检查x:Bind的路径是否正确

## 性能问题

### 问题1: 应用启动慢

**症状**: 应用启动需要很长时间

**解决方案**:

1. **优化启动代码**
   - 将耗时操作移到后台线程
   - 使用async/await避免阻塞UI线程

2. **延迟加载**
   - 使用Lazy加载非关键资源
   - 实现按需加载页面

3. **减少依赖**
   - 移除不必要的NuGet包
   - 使用条件编译排除调试代码

### 问题2: 内存占用高

**症状**: 应用占用大量内存

**解决方案**:

1. **检查内存泄漏**
   - 使用Visual Studio内存分析工具
   - 查找未释放的对象

2. **优化集合**
   - 使用ObservableCollection而非List
   - 及时清理不再使用的项

3. **释放资源**
   - 实现IDisposable接口
   - 在页面卸载时释放资源

### 问题3: UI卡顿

**症状**: 界面响应缓慢

**解决方案**:

1. **启用虚拟化**
   - 在ListView/GridView中启用虚拟化
   - 使用ItemsStackPanel而非StackPanel

2. **减少重绘**
   - 避免频繁更新UI
   - 使用节流/防抖技术

3. **优化动画**
   - 使用合成动画而非故事板
   - 减少同时运行的动画数量

## 网络问题

### 问题1: 实时消息不更新

**症状**: 聊天消息不实时显示

**解决方案**:

1. **检查SignalR连接**
   - 在ChatService中添加连接状态监听
   - 处理断线重连逻辑

2. **验证Hub URL**
   - 检查appsettings.json中的SignalRHubUrl
   - 确保URL可访问

3. **实现重连机制**
   ```csharp
   _connection.Closed += async (error) =>
   {
       await Task.Delay(new Random().Next(0, 5) * 1000);
       await _connection.StartAsync();
   };
   ```

### 问题2: 文件上传失败

**症状**: 无法上传文件到服务器

**解决方案**:

1. **检查文件大小限制**
   - 确认服务器允许的文件大小
   - 压缩大文件或分块上传

2. **验证文件类型**
   - 确保上传的文件类型被允许
   - 检查文件扩展名

3. **添加进度显示**
   - 使用IProgress<T>接口显示上传进度
   - 提供取消上传的选项

## UI问题

### 问题1: 主题不生效

**症状**: 切换主题后UI颜色不变

**解决方案**:

1. **检查主题设置**
   - 确保正确设置RequestedTheme
   - 验证主题资源定义

2. **强制刷新**
   - 重新加载页面或应用
   - 清除应用缓存

3. **验证资源字典**
   - 确保主题资源在App.xaml中正确合并
   - 检查资源键名是否正确

### 问题2: 控件显示异常

**症状**: 控件显示不正确或缺失

**解决方案**:

1. **检查控件样式**
   - 确保自定义样式不覆盖默认样式
   - 使用BasedOn继承默认样式

2. **验证布局**
   - 检查Grid/StackPanel等布局容器
   - 确保控件在可见区域内

3. **检查数据模板**
   - 验证DataTemplate中的绑定
   - 确保模板正确应用

## 调试技巧

### 1. 使用输出窗口

在Visual Studio中查看调试输出：

```
调试 > 窗口 > 输出
```

选择"调试"或"生成"输出源。

### 2. 使用实时可视化树

查看运行时UI结构：

```
调试 > 窗口 > 实时可视化树
```

可以实时修改属性值查看效果。

### 3. 使用诊断工具

Visual Studio提供的诊断工具：

- **CPU使用率**: 分析CPU性能
- **内存使用**: 检测内存泄漏
- **GPU使用**: 分析GPU性能

### 4. 添加日志记录

在代码中添加日志：

```csharp
System.Diagnostics.Debug.WriteLine($"Debug info: {info}");
```

在输出窗口中查看日志。

### 5. 使用断点

设置条件断点：

1. 在代码行号左侧右键
2. 选择"条件"
3. 输入条件表达式

断点只在条件为真时触发。
