## 1. 项目概述

### 1.1 项目简介

HMNL直播系统是一个综合性的内容管理与直播平台，集成了直播频道管理、社区论坛、世界观创作、AI对话等多种功能模块。平台采用现代化的前端技术栈，遵循 Material Design 3 设计规范，提供流畅的用户体验。

### 1.2 核心特性

- **多端适配**：响应式设计，支持桌面端和移动端
- **实时通信**：支持实时聊天、直播弹幕等实时交互功能
- **内容创作**：支持帖子发布、世界观创作、直播页面定制
- **用户体系**：完整的用户认证、权限管理、实名认证体系
- **直播系统**：支持RTMP推流、HLS/FLV播放、频道管理

### 1.3 设计规范

- 遵循 Material Design 3 (MD3) 完整规范
- 配色方案基于全局 CSS Custom Properties
- 图标系统使用 Material Icons
- 支持 WCAG 2.1 AA 级无障碍标准

### 3.1 用户认证模块

#### 功能描述
提供多种认证方式，包括用户名密码登录、手机验证码登录、人脸识别登录，以及访客模式。

#### 功能列表

| 功能名称 | 功能描述 | 权限要求 |
|---------|---------|---------|
| 用户名密码登录 | 使用用户名/数字ID + 密码登录 | 无 |
| 手机验证码登录 | 使用手机号 + 短信验证码登录 | 无 |
| 人脸识别登录 | 使用已注册的人脸信息登录 | 需预先注册人脸 |
| 用户注册 | 创建新用户账户 | 无 |
| 访客模式 | 无需登录浏览公开内容 | 无 |
| 实名认证 | 身份证二要素验证 | 已登录用户 |

#### 输入参数

**用户名密码登录**
```typescript
interface SignInParams {
  userId: string;     // 用户名或数字ID
  password: string;   // 密码
}
```

**手机验证码登录**
```typescript
interface PhoneSignInParams {
  phone: string;      // 手机号
  code: string;       // 验证码
}
```

**用户注册**
```typescript
interface SignUpParams {
  username: string;   // 用户名
  password: string;   // 密码
  phone?: string;     // 手机号（可选）
}
```

**实名认证**
```typescript
interface RealNameVerificationParams {
  name: string;       // 真实姓名
  idCard: string;     // 身份证号码
}
```

#### 业务规则

1. **用户名规则**
   - 用户名必须唯一
   - 不允许使用已存在的用户名注册

2. **密码规则**
   - 密码需满足Supabase密码强度要求

3. **实名认证规则**
   - 调用第三方API验证姓名与身份证号匹配
   - 认证成功后保存姓名和身份证后4位
   - 完整身份证号仅超级管理员可查看

4. **人脸登录规则**
   - 用户需先注册人脸信息
   - 使用Magic Link方式实现无密码登录

### 3.2 直播频道模块

#### 功能描述
提供直播频道的创建、管理、播放功能，支持RTMP推流和HLS/FLV播放。

#### 功能列表

| 功能名称 | 功能描述 | 权限要求 |
|---------|---------|---------|
| 频道列表 | 浏览所有活跃频道 | 无 |
| 频道播放 | 观看直播内容 | 无 |
| 频道创建 | 创建新的直播频道 | 入驻主播 |
| 频道管理 | 编辑频道信息、管理直播状态 | 频道所有者/管理员 |
| 直播状态 | 查看频道直播状态 | 无 |
| 点赞功能 | 为频道点赞 | 已登录用户 |

#### 输入参数

**获取直播间URL**
```typescript
interface LiveRoomUrlRequest {
  channel_id?: string;    // 频道ID
  channel_url?: string;   // 频道URL标识
  stream_id?: string;     // 推流ID
}
```

**获取播放器URL**
```typescript
interface PlayerUrlRequest {
  channel_id?: string;    // 频道ID
  channel_url?: string;   // 频道URL标识
  format?: 'hls' | 'flv' | 'auto';  // 播放格式
  quality?: 'high' | 'medium' | 'low' | 'auto'; // 画质
}
```

**获取频道信息**
```typescript
interface ChannelInfoRequest {
  channel_id?: string;    // 频道ID
  channel_url?: string;   // 频道URL标识
  include_stats?: boolean; // 是否包含统计数据
}
```

#### 输出参数

**直播间URL响应**
```typescript
interface LiveRoomUrlData {
  channel_id: string;
  channel_name: string;
  channel_url: string;
  stream_id: string;
  rtmp_push_url: string;    // RTMP推流地址
  rtmp_push_key: string;    // 推流密钥
  hls_play_url: string;     // HLS播放地址
  flv_play_url: string;     // FLV播放地址
  web_play_url: string;     // 网页播放地址
  is_live: boolean;
  created_at: string;
}
```

**频道信息响应**
```typescript
interface ChannelInfoData {
  id: string;
  name: string;
  description: string | null;
  cover_image: string | null;
  channel_url: string;
  stream_id: string;
  is_live: boolean;
  is_active: boolean;
  like_count: number;
  viewer_count?: number;
  owner?: {
    id: string;
    username: string;
    avatar_url: string | null;
    is_verified: boolean;
  };
  stats?: {
    total_views: number;
    total_likes: number;
    followers: number;
  };
  created_at: string;
  updated_at: string;
}
```

#### 业务规则

1. **频道创建规则**
   - 用户需先成为入驻主播
   - 系统自动生成唯一的频道URL
   - 频道URL不可重复

2. **直播状态管理**
   - 推流开始时自动更新为直播中
   - 推流结束时自动更新为未直播

3. **频道访问规则**
   - 已停用的频道无法访问
   - 非活跃频道不在列表显示

try {
